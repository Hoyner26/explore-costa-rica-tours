# Decisiones UI/UX

## Objetivo
Diseñar una aplicación clara, intuitiva y responsive para que los usuarios puedan explorar y reservar tours fácilmente tanto en web como en móvil.

## Decisiones principales
- Se utilizará una navegación simple y clara.
- Las pantallas mostrarán la información más importante primero.
- Los formularios tendrán validaciones visibles.
- Se priorizará un diseño responsive para web y móvil.
- Se implementará modo claro y oscuro con persistencia.
- Se usarán componentes consistentes para mejorar la experiencia del usuario.

## Principios aplicados
- Visibilidad del estado del sistema
- Prevención de errores
- Consistencia visual
- Accesibilidad básica
- Diseño responsive

## Referencia visual
La aplicación toma como referencia experiencias similares a plataformas de exploración turística, buscando una interfaz moderna, limpia y fácil de entender.
